# Trainpack Format

Vocabulary V2 trainpacks are official speaking-action packages. This repository has no shipped legacy release, so the format is defined directly around the V2 model.

## Catalog

`manifests/latest.json` is the app entry point. It contains:

- `schemaVersion`: fixed at `1` for the physical file contract.
- `catalogVersion`: date version, currently `2026.06.03`.
- `minAppVersion`: currently `1.2.0`.
- `packs[]`: official pack entries.

Each pack entry must include:

- identity: `id`, `title`, `description`, `packType`, `version`
- sizing: `unitCount`, `itemCount`, `pronunciationReferenceCount`, `estimatedMinutes`, `sizeBytes`
- integrity: `sha256`, `fileName`, `urls`
- learning metadata: `seriesId`, `seriesTitle`, `seriesOrder`, `learningPath`, `recommendedNextPackIds`, `companionPackIds`
- queue metadata: `dailyQueueRoles`

`packType` is currently only `SpeakingActs`.

## Package Files

Every `.trainpack` is a zip with exactly four files:

```text
manifest.json
units.json
items.jsonl
pronunciation_references.jsonl
```

## Unit / SpeakingAct

Each unit is a speaking action. Required fields:

- `unitId`
- `speakingActId`
- `order`
- `title`
- `communicativeGoal`
- `scene`
- `sceneCoverage`
- `register`
- `difficulty`
- `tags`
- `activationPrompts`
- `itemIds`
- `pronunciationFocus`
- `taskHints`

`taskHints` must support daily queue generation:

- `defaultRole`
- `successCriteria`
- `correctionFocus`
- `retryPrompts`
- `variantPrompts`
- `recommendedTaskTypes`
- `transferPrompts`
- `weakPointPrompts`

## Item / Move

Each item is one concrete move under a speaking action. Supported item types:

- `Chunk`
- `SentencePattern`
- `DialogueTurn`
- `RepairStrategy`
- `PronunciationTarget`

The app may show different moves as fallback, primary, natural, repair, transfer, or pronunciation-only tasks, but formal text still comes only from this official pack data.

## Pronunciation References

`pronunciation_references.jsonl` is generated by the build tools and must not be hand-authored.

Each line includes:

- `id`, `itemId`, `unitId`, `unitTitle`
- `targetType`, `targetText`, `dialect`
- `phonemes`
- `words[]`
- `words[].lexicalStress`
- top-level `lexicalStress`
- top-level `sentenceStress`
- `source = PackVerifiedOnline`
- `reviewStatus = PackVerified`
- `provenance`

The app should only assess the current task's `pronunciationFocus`. It must not turn these references into broad automatic correction for arbitrary user-created content.

## Validation

```bash
.venv-pronunciation/bin/python tools/release_trainpacks.py
```

The release command builds all source packs, writes the catalog, and validates every generated trainpack against `manifests/latest.json`.
